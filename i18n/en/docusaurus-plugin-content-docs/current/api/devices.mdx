## <Search size={20} style={{display: 'inline', verticalAlign: 'middle', marginRight: '0.5rem'}} color="var(--ifm-color-primary)" /> Detailed Data Description

### 1. Basic Identification

```json
{
  "id": 1234567890123456,
  "identifier": "abc1-def2-ghi3-jkl4",
  "title": "Charger 1",
  "machine_id": "MB2024001",
  "serial_number": "MB-12345"
}
```

| Field | Type | Description |
|------|-----|-------|
| `id` | number | Unique numeric ID in the system |
| `identifier` | string | Text identifier (used in API) |
| `title` | string | Your station name |
| `machine_id` | string | Machine number |
| `serial_number` | string | Serial number |

---

### 2. Device Status

```json
{
  "status": "paired",
  "state": "ready",
  "firmware_status": "reflashed",
  "pairing_mode": "wifi"
}
```

#### What do individual states mean?

**Status** - Pairing state:
- `new` - New, unpaired device
- `pairing` - Pairing in progress
- `paired` - Successfully paired
- `free` - Available for use

**State** - Current state:
- `not_connected` - Not connected to internet
- `disconnected` - Disconnected (was previously online)
- `ready` - Online and ready
- `charging` - Currently charging a vehicle

**Firmware Status**:
- `new` - Default firmware
- `flashed` - Firmware uploaded
- `reflashed` - Firmware updated

---

### 3. Product Information

```json
{
  "product": {
    "id": 1682081987312165,
    "title": "MyBox Home",
    "version": "7.3",
    "mcu": "ESP32",
    "status": "production",
    "abbreviation": "17",
    "type": "R"
  }

```

| Field | Meaning |
|------|---------|
| `title` | Charger model (Home, Post, Profi...) |
| `version` | Product version |
| `mcu` | Processor type |
| `status` | Production status |

## <Lightbulb size={20} style={{display: 'inline', verticalAlign: 'middle', marginRight: '0.5rem'}} color="var(--ifm-color-primary)" /> Practical Use of This Data

### 1. Availability Check
Check `state`:
- `ready` = Device is online
- `disconnected` = Device is offline

### 2. Model Identification
From `product.title` you can determine the charger type:
- MyBox Home - home charger
- MyBox Post - pedestal, 2 connectors
- MyBox Profi - professional

### 3. Firmware Currency
Compare `firmware_name` with the latest version

### 4. Localization
GPS coordinates in `location` can be used for:
- Display on map
- Distance calculation
- Navigation

## <HelpCircle size={20} style={{display: 'inline', verticalAlign: 'middle', marginRight: '0.5rem'}} color="var(--ifm-color-primary)" /> Frequently Asked Questions

**Q: Why is `user_title` null?**
A: User hasn't set a custom name. The default `title` is used.

**Q: What's the difference between `state: disconnected` vs `not_connected`?**
A: `disconnected` = was online but lost connection
`not_connected` = has never been online

**Q: Can I change the device name?**
A: Yes, but that requires a POST endpoint (not part of the basic API).
